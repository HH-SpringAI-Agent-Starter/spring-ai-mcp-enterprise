# MCP Enterprise V0.11.0 — MCP 2026-07-28 规范全面适配

> 🔥 2026-07-19 | Spring AI MCP Enterprise 成为 Java 生态首个全面适配 MCP 2026-07-28 规范的企业级框架

## 🏭 核心变化

MCP 2026-07-28 是 MCP 协议史上最大修订，从"工具调用连接协议"升级为"生产级 Agent 基础设施"。V0.11 完成以下全面适配：

### 1. 无状态核心 ✅
- 每个请求自包含（无需 Session 绑定）
- 支持 K8s / Cloud Run 弹性伸缩
- streamable-http transport 完整支持

### 2. 能力发现 ✅
- 新增 `tools/discover` 端点（单工具深度能力查询）
- 新增 `server/discover` 端点（Server 全量能力清单）
- 每个工具返回完整 inputSchema/outputSchema/缓存策略/速率限制/调用示例
- 支持网关路由、客户端缓存、平台限流和审计

### 3. W3C Trace Context ✅
- `traceparent` / `tracestate` 标准头部
- 全链路追踪：从 AI 客户端 → MCP Gateway → Tool → DB
- 兼容 OpenTelemetry / Jaeger / Zipkin

### 4. 完整 JSON Schema 2020-12 ✅
- 所有工具 inputSchema 升级到 JSON Schema 2020-12
- 支持 `$ref`、`oneOf`、`anyOf` 等高级组合
- 结构化输出交付

### 5. Extensions 一等公民 ✅
- 声明支持的 Extension 命名空间
- 支持第三方扩展协议独立演进

### 6. Tasks 长任务支持 ✅
- 新增 `tasks/create` 端点
- 异步任务提交 → 返回 taskId → 轮询状态
- 最多 5 分钟超时

### 7. 企业授权加固 ✅（已有）
- RBAC 三级权限
- JWT / OAuth2 / API Key / SSO
- 审计日志 + 速率限制

## 🚀 升级指南

### 旧版客户端（2025-03-26 协议）

向后兼容。V0.11 自动检测客户端声明版本，返回对应能力声明：

```json
// 客户端声明 protocolVersion: "2025-03-26" → 返回旧版 SSE 端点能力
// 客户端声明 protocolVersion: "2026-07-28" → 返回新版无状态核心能力
```

### 新版客户端（2026-07-28 协议）

```bash
# 无状态 HTTP 调用（推荐）
curl -X POST http://localhost:8080/api/mcp/message \
  -H "Content-Type: application/json" \
  -H "traceparent: 00-{trace-id}-{span-id}-01" \
  -d '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2026-07-28"}}'

# 能力发现
curl -X POST http://localhost:8080/api/mcp/message \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":2,"method":"server/discover","params":{}}'
```
